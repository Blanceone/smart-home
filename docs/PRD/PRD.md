## 一、产品背景

### 1.1 市场背景
随着智能家居市场的快速发展，越来越多的年轻人希望打造智能化的居住环境。然而，面对市场上琳琅满目的智能家居产品，用户往往面临以下痛点：
- 不清楚自己的户型适合哪些智能设备
- 不了解不同品牌产品的性价比差异
- 难以在预算范围内做出最优选择
- 缺乏系统性的方案设计能力

### 1.2 产品定位
一款面向年轻用户的智能家居方案设计工具，通过AI技术为用户量身定制智能家居方案，帮助用户在预算范围内实现居住空间的智能化升级。

### 1.3 产品目标
- 降低智能家居入门门槛
- 提供个性化、可落地的智能方案
- 连接用户与电商平台，实现方案到购买的闭环

## 二、用户画像

### 2.1 核心用户群体

| 用户类型 | 特征描述 | 核心诉求 |
|----------|----------|----------|
| 新房装修族 | 25-35岁，刚购房，预算相对充足 | 全屋智能方案，一站式解决 |
| 租房改造族 | 22-30岁，租房居住，预算有限 | 高性价比、易拆卸的智能设备 |
| 科技爱好者 | 20-35岁，对智能产品有了解 | 最新产品、品牌偏好明确 |
| 家庭升级族 | 30-45岁，已有住房，想升级 | 渐进式改造，兼容现有设备 |

### 2.2 典型用户故事

**用户故事1：新房装修族 - 小王**
> 小王（28岁）刚买了第一套房，90平米两室一厅。他对智能家居很感兴趣，但不知道从何下手。他希望有一个工具能根据他的户型和生活习惯，推荐一套完整的智能家居方案，预算在1-2万元。

**用户故事2：租房改造族 - 小李**
> 小李（25岁）独自租住一间30平米单身公寓，预算只有2000元。她希望添加一些基础的智能设备（智能灯、智能插座等），但担心搬家时设备难以带走。

#### 3.2.5 AI方案生成服务（后端）

**处理流程：**
1. 接收用户请求数据（户型+问卷+偏好）
2. 构建结构化提示词
3. 调用DeepSeek API生成方案
4. 解析AI返回的方案数据
5. 返回结构化方案

**提示词模板：**
```
你是一位专业的智能家居方案设计师。请根据以下用户信息，为其设计一套智能家居方案。

## 用户户型信息
- 总面积：{totalArea}平方米
- 房间列表：{rooms}

## 用户需求信息
- 居住情况：{livingStatus}
- 常住人数：{residentCount}人
- 是否有老人：{hasElderly}
- 是否有儿童：{hasChildren}
- 是否有宠物：{hasPets}
- 期望场景：{preferredScenarios}
- 作息习惯：{sleepPattern}

## 输出要求
请以JSON格式输出方案，包含以下字段：
1. schemeName: 方案名称
2. schemeDescription: 方案详细描述（200字以内）
3. devices: 设备列表数组，每个设备包含：
   - deviceType: 设备类型
   - deviceName: 设备名称
   - room: 放置房间
   - quantity: 数量
   - reason: 推荐理由

请确保方案合理、实用，设备数量适中。
```

#### 3.2.6 产品匹配服务（后端）

**匹配算法逻辑：**
```
输入：AI方案设备列表 + 用户预算 + 品牌偏好 + 商品数据库
输出：匹配后的具体商品列表

匹配步骤：
1. 按设备类型筛选商品
2. 按品牌偏好排序（优先匹配偏好品牌）
3. 按价格区间筛选（预算约束）
4. 按评分/销量排序
5. 选择Top N商品
6. 计算总价，确保不超预算
```

## 七、数据实体关系

### 7.1 用户模式说明

**匿名使用模式**：用户无需注册登录，打开APP即可使用全部功能。

- 用户数据通过**设备唯一标识（DeviceId）**关联
- DeviceId在首次启动时自动生成并存储在本地
- 用户数据保存在本地设备，支持云端备份（可选）
- 用户可随时清除本地数据

| 项目 | 说明 |

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|


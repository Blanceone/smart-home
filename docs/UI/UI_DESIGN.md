## 一、设计概述

### 1.1 设计理念

**极简智能，温暖生活**

- **极简主义**：减少视觉噪音，聚焦核心功能
- **科技温度**：智能家居不是冷冰冰的技术，而是温暖的生活方式
- **流畅体验**：细腻的动效让等待也成为享受

### 1.2 架构对齐声明

> 本设计严格遵循 [../ARCH/ARCHITECTURE.md](../ARCH/ARCHITECTURE.md) v1.2 定义的模块边界：
> 
> **前端模块**：
> - **户型管理模块** → 本地存储户型数据，服务端图片解析（可选）
> - **问卷偏好模块** → 本地存储问卷/偏好数据，服务端方案生成
> - **方案展示模块** → 本地存储方案数据，服务端商品匹配
> - **本地数据管理模块** → 纯本地操作，历史方案、数据清除、数据导出
> - **日志管理模块** → 本地日志记录、日志上传、日志清理
> 
> **后端模块**：
> - **AI方案服务** → 无状态，不存储用户数据
> - **商品服务** → 仅存储商品数据，不存储用户数据
> - **日志收集服务** → 接收、存储、分析客户端日志

### 1.3 架构变更历史

| 变更项 | v1.0 | v1.1 | v1.2/v1.3 |
|--------|------|------|-----------|
| 用户体系 | 需要登录注册 | **设备级匿名使用** | 设备级匿名使用 |

| 本地数据管理 | - | **新增本地数据管理模块** | 本地数据管理模块 |

## 三、交互规范

### 3.1 动画原则

- **自然流畅**：遵循物理规律，使用缓动曲线
- **目的明确**：动画服务于功能，不为动画而动画
- **性能优先**：避免卡顿，优先使用 transform 和 opacity

### 3.2 缓动曲线

| 曲线名称 | CSS值 | 用途 |
|----------|-------|------|
| ease-out | `cubic-bezier(0, 0, 0.2, 1)` | 元素进入 |
| ease-in | `cubic-bezier(0.4, 0, 1, 1)` | 元素退出 |
| ease-in-out | `cubic-bezier(0.4, 0, 0.2, 1)` | 状态切换 |
| spring | `cubic-bezier(0.34, 1.56, 0.64, 1)` | 弹性效果 |

### 3.3 时长规范

| 动画类型 | 时长 | 说明 |
|----------|------|------|
| 微交互 | 150ms | 按钮点击、图标切换 |
| 常规过渡 | 250ms | 页面切换、展开收起 |
| 复杂动画 | 400ms | 列表加载、模态框 |
| 强调动画 | 600ms | 引导关注、成功反馈 |

### 3.4 核心交互动画

#### 3.4.1 页面转场

```
页面进入：
- 起始状态：translateX(100%)，opacity: 0
- 结束状态：translateX(0)，opacity: 1
- 时长：300ms
- 曲线：ease-out

页面退出：
- 起始状态：translateX(0)，opacity: 1
- 结束状态：translateX(-30%)，opacity: 0
- 时长：250ms
- 曲线：ease-in
```

#### 3.4.2 按钮交互

```
点击反馈：
- 按下：scale(0.97)，时长 100ms
- 释放：scale(1)，时长 150ms，曲线 spring
- 禁用：opacity: 0.5，无动画
```

#### 3.4.3 卡片展开

```
展开动画：
- 高度：0 → auto
- 透明度：0 → 1
- 时长：300ms
- 曲线：ease-out
- 内部元素：stagger 延迟 50ms 依次进入
```

#### 3.4.4 骨架屏动画

```
骨架闪烁：
- 背景：linear-gradient(90deg, #F3F4F6 25%, #E5E7EB 50%, #F3F4F6 75%)
- 动画：shimmer 1.5s infinite
- 背景-size：200% 100%
```

#### 3.4.5 左滑删除

```
左滑删除：
- 向左滑动：translateX(-80px)，显示删除按钮
- 删除按钮：红色背景 #EF4444，白色图标
- 点击删除：弹出确认对话框
- 确认后：卡片向左滑出屏幕，高度收缩为0
```

### 3.5 等待动画设计

#### 方案生成等待动画

| 阶段 | 动画内容 | 时长 | 提示文案 |
|------|----------|------|----------|
| 数据上传 | 进度条填充 | 1-3s | "正在上传您的数据..." |
| AI分析 | 脉冲圆环 + 粒子效果 | 5-15s | "AI正在为您量身定制方案..." |
| 产品匹配 | 齿轮旋转 + 勾选动画 | 2-5s | "正在匹配最适合您的产品..." |

**文案轮播规则**：每 3 秒切换一次提示文案

**动画参数**：
```
脉冲圆环：
- 外圈：scale(1 → 1.5)，opacity(1 → 0)，时长 2000ms，循环
- 内圈：延迟 500ms 后执行相同动画

粒子效果：
- 数量：20 个粒子
- 运动：从中心向外扩散
- 时长：1500ms
- 曲线：ease-out
```

```
页面加载：
1. 扫描本地日志目录
2. 统计日志文件数量和大小
3. 解析日志级别分布
4. 计算日志时间范围
```

**上传流程**：
```
1. 检查日志文件大小（≤5MB）
2. 超过限制 → 提示清理
3. 未超过限制 → 压缩日志文件
4. 调用 POST /logs/upload 接口
5. 显示上传进度
6. 上传成功 → 显示成功页面
7. 上传失败 → 显示错误信息，提供重试
```

**按钮样式**：
- 上传按钮：渐变背景（智能蓝），高度56px，带图标
- 清理按钮：次要按钮样式，中性灰色

**交互逻辑**：
- 点击"上传日志到服务器" → 检查大小 → 开始上传
- 点击"清理日志后上传" → 弹出清理确认 → 清理后上传
- 上传中点击"取消上传" → 取消上传请求
- 上传失败点击"重新上传" → 重新开始上传
